# Future Directions

Broad decided directions. These are committed visions, not implementation plans.

## Multi-User Instances
Multiple people sharing one Kernos instance with trust tiers. Family members, team members, or clients — each with appropriate access levels and visibility boundaries.

## Web Interface
A web UI for real users. Chat interface, settings, capability management, covenant editing, space navigation. The same docs/ that the agent reads will be published as user-facing reference.

## PIE Expansion (Proactive Intelligence Engine)
Observe everything — not just foresight signals. Collision detection (conflicting commitments), anomaly detection (unusual patterns), opportunity detection (connections the user would miss), pattern detection (recurring behaviors worth automating). Multiple evaluation passes beyond the current time pass.

## Multi-LLM Routing and Cost Optimization
Route different tasks to different models based on complexity and cost. Simple routing and extraction to cheap models, complex reasoning to capable models. Dynamic model selection per-call.

## Customer-Facing Spaces
Spaces where the agent interacts with the user's customers or clients, with appropriate disclosure and boundary management.

## Standing Orders and Event-Triggered Automation
Full execution of stored automation rules. "Every Monday send me a summary" actually fires on Monday. "When someone emails about invoices, flag it" actually monitors and flags.

## Unified Capability Registry
Complete the vision: defaults and user-added tools are indistinguishable in the registry. No special-casing for pre-installed capabilities. Everything managed through one interface.
